# 🍕 義次元腦洞大開：校園瑪瑪咪呀合成大戰 - UI 規格書

本規格書基於《企劃.md》、《詳細劇本與關卡設定.md》與《技術規格.md》，詳細規劃遊戲中所有網頁介面的設計邏輯、操作流程與組件規格。

---

## 一、 共用 UI 元件與全域設計規範 (Global Settings)

### 1. 視覺與技術基底
* **響應式設計 (RWD)**：優先以手機版面 (Mobile-first) 設計，並完美適配平板 (Tablet) 尺寸。
* **整體風格**：參照提供的 `ui 參考.png`，風格需活潑、童趣，並融合「迷因 (Meme/Brainrot)」元素（如誇張的表情、無厘頭的圖示）。
* **字型設定**：統一使用「清松手寫體 `JasonHandwriting2-Regular.woff2`」，並須進行字體子集壓縮以優化載入速度。
* **技術框架**：React + Tailwind CSS，後端與狀態管理使用 Firebase (Firestore / Storage)。

### 2. 共用組件 (Shared Components)
* **Bottom Navigation Bar (底部導覽列) / FAB (浮動操作按鈕)**：
  * **[背包/任務狀態]**：點擊切換至背包頁籤，檢視目前已收集的食材、天線碎片與線索。
  * **[掃描 QR Code]**：長駐於畫面中央或底部明顯處，點擊即開啟相機掃描 QR Code（進入關卡或合成器）。
* **Audio Player (語音播放模組)**：
  * 背景自動播放：進入角色關卡時先播放本機音檔（如角色大喊自己的設定口頭禪或名字）。
  * TTS (Text-to-Speech)：接續朗讀角色的對話/題目。聲音參數需依照角色性格動態代入（如 Ninja 聲線、暴躁大叔聲線等）。
* **通知與彈窗 (Alert & Modal)**：
  * 用於顯示「獲得道具」、「密碼錯誤」、「關卡冷卻中 (Cooldown)」的幽默嘲諷畫面。
* **Card Component (虛擬寶物卡牌)**：
  * 精緻的卡牌邊框，內含：食材插圖、幽默物品描述。

## 二、 頁面總覽與導覽流向 (Page Flow)

1. **入口設定**：
   `[報到/登入頁面]` $\rightarrow$ 輸入小隊名稱/編號 $\rightarrow$ 進入 `[主畫面/背包頁面]`
2. **冒險探索**：
   在 `[主畫面/背包頁面]` 點擊掃描 $\rightarrow$ 掃描實體 QR Code $\rightarrow$ 進入 `[關卡過場/角色對話頁面]` $\rightarrow$ 點擊挑戰 $\rightarrow$ `[關卡互動任務頁面]`
3. **結算邏輯**：
   挑戰成功 $\rightarrow$ `[結算獎勵頁面]` (領取食材、線索) $\rightarrow$ 返回 `[主畫面/背包頁面]`
   挑戰失敗 $\rightarrow$ `[冷卻嘲諷頁面]` $\rightarrow$ 返回 `[主畫面/背包頁面]` (該關卡鎖定 X 分鐘)
4. **合作加工階段**：
   倒數結束/收集完畢 $\rightarrow$ 掃描實體合成器 $\rightarrow$ `[合成前置判定頁]` (檢查 5 種食材，不足則提示配對) $\rightarrow$ `[線上合作合成頁面]` $\rightarrow$ 拖曳食材完成 $\rightarrow$ `[結局與證書頁面]`

## 三、 各頁面詳細規格與按鈕功能

### 1. 入口與報到頁面 (Login / Entry Page)
* **功能目的**：讓小朋友以小隊為單位登入，與 Firebase 綁定狀態。
* **畫面元素**：
  * 遊戲大 Logo 與迷因風格背景。
  * `[輸入小隊編號/名稱]`：文字輸入框。
  * `[出發拯救校園！]`：超大型確認按鈕。

### 2. 主畫面與背包頁面 (Dashboard & Backpack)
* **功能目的**：遊戲進行中的核心首頁，檢視進度與手邊資源。
* **畫面元素**：
  * **頂部狀態列**：顯示小隊名稱、剩餘時間倒數計時器。
  * **收集進度區**：用剪影或灰階圖示顯示終極披薩所需的 5 項材料，獲得的會亮起並變為卡牌樣式。
  * **情報與線索區**：顯示目前拿到的提示文字（例如「水球鱷魚知道起司在哪裡」）。
  * **操作按鈕**：
    * `[開啟掃描器]`：呼叫相機。
    * `[背包卡牌]`：點擊卡牌可放大觀看搞笑敘述。

### 3. 通用關卡進入與對話頁面 (Character Intro Page)
* **功能目的**：呈現該關卡的背景故事，播放語音。
* **畫面元素**：
  * **角色大圖**：動態或立繪呈現（由 Firebase Storage 讀取無背圖片）。
  * **對話框 (Dialog Box)**：類似 RPG 遊戲，文字具備打字機特效。
  * **語音控制**：自動播放音效與 TTS，提供 `[重播對話]` 按鈕。
  * **操作按鈕**：
    * `[接受挑戰！]`：點擊進入具體任務頁面。

### 4. 關卡專屬互動任務頁面 (Task Pages)
因每關互動不同，UI 須針對各別關卡實作：

* **🔥 關卡 1 (Cappuccino Assassino)：忍術排序**
  * **UI**：畫面顯示三個忍術按鈕（☕咖啡杯、🔥火焰、⚔️武士刀）。底部有 3 個空槽位。
  * **操作**：點擊圖示填入槽位，具備 `[清除]` 與 `[發動忍術！]`(確認) 按鈕。
* **🩰 關卡 2 (Ballerina Cappuccina)：保險箱密碼鎖**
  * **UI**：大型 4 位數實體數字鍵盤風格。
  * **操作**：點擊輸入 4 碼數字，具備 `[解鎖]` 按鈕。
* **🐸 關卡 3 (Brr Brr Patapim)：螢幕狂點挑戰**
  * **UI**：畫面中央有一顆大氣球，上方有顯眼的 10 秒倒數計時器，下方為進度條 (0/50)。
  * **操作**：氣球本身為一個巨大按鈕 `[狂點這裡！]`，點擊會發出充氣音效與動畫震動。
* **🐊 關卡 4 (Bombardilo Crocodilo)：水球拖曳排序**
  * **UI**：四顆不同顏色的水球圖示排在待選區，下方為 1~4 名的排序空槽。
  * **操作**：支援 **Drag and Drop (拖曳)** 或點選排序。底部有 `[送出答案避開炸彈]` 按鈕。
* **🐘 關卡 5 (Lirili Larila)：時間輸入解謎**
  * **UI**：上方列出前四道鎖的時間提示。下方為仿數位時鐘輸入欄位 (HH:MM)。
  * **操作**：使用下拉滾輪或鍵盤輸入 `07:45`，點擊 `[重啟鬧鐘]`。
* **🪵 關卡 6 (Tung Tung Tung Sahur)：大猩猩認證**
  * **UI**：呼叫相機介面或密碼輸入介面。
  * **操作**：`[拍照上傳]` 或 `[輸入關主認證碼]` 以確認通關，並且在上傳的圖片當中加入疊圖層，右上角有這個「Tung Tung Tung Sahur」的圖片
* **🦈 關卡 7 (Tralalero Tralala)：跨小隊天線共鳴**
  * **UI**：顯示自身隊伍天線顏色（紅或藍）及持有的一半密碼。畫面中央有 6 碼的大型輸入框。
  * **操作**：與友隊交流後，雙方各自在自己的畫面上輸入 6 碼完整字串，點選 `[破解防護罩]`。

### 5. 關卡結算頁面 (Result & Cooldown Page)
* **成功 UI**：撒花特效、展示抽卡動畫（獲得食材卡牌與新線索），`[放入背包並繼續探索]`。
* **失敗 UI**：角色嘲諷圖，顯示「大腦過熱發爐中」，下方顯示冷卻倒數計時器（如 02:59），提示 `[先去其他地方轉轉]`。

### 6. 終極合成大廳與合作烤箱 (Co-op Synthesis Room)
* **條件判定**：掃描合成協作站 QR Code 後，系統會檢查是否集齊 5 種食材（麵粉、水、番茄、起司、羅勒葉）；若食材不足，UI 會進入**「尋找盟友雷達介面」**，顯示：「請尋找 X 號小隊合作」。
* **合作烤箱 UI**：
  * 畫面分為兩半：「你的食材區」與中央大型「披薩烤盤區」。
  * 支援拖曳 (Drag & Drop) 或者點擊 `[投入]`，將背包中擁有的食材丟入畫面中央。
  * 使用 Firebase 實時同步技術 (Realtime Sync)，當我方投入起司時，對方畫面也會即時顯示起司飛入烤爐。
* **防呆機制**：帶有隱藏的 `[呼叫關主救援]` 或判定無隊伍可合作時彈出 `[向全校求助]` 虛擬按鈕。

### 7. 結局與數位證書頁面 (Victory Page)
* **功能目的**：總結遊戲，發放紀念獎勵。
* **畫面元素**：
  * **成績單**：通關時間、幫助了幾位角色。
  * **數位證書 (Hero Certificate)**：動態生成的獎狀，印有小隊名稱與所有角色合照。
  * 具備 `[儲存證書圖片]` 或 `[分享]` 的按鈕。

## 四、 導覽功能的選擇邏輯 (Navigation & Logic)

1. **SPA (單頁應用程式) 路由**：
   不造成網頁整頁重整（以維持語音與狀態的一致性），全面使用 React Router 進行切換。
2. **防作弊與掃描邏輯**：
   * 每個關卡的 QR Code 皆攜帶一組 Token 或獨特的 ID 參數（例如 `?level=capu&token=xyz`）。
   * 不能直接點擊連結跳轉關卡，只能透過相機掃描進入，或由遊戲主頁面控制的合法路由進行。
3. **雲端存檔邏輯**：
   * 每通關一關，立即同步結果至 Firebase，確保網頁重整、手機閃退後，關卡進度與背包物品不會遺失。

## 五、 UI 開發用 Prompt (指導 AI 或設計師)

> **【請將以下 Prompt 提供給 AI 程式碼生成助手，以產生符合規格的 UI 程式碼】**
>
> **System Role:** 你是一位頂尖的前端 UI/UX 專家與 React 開發者，擅長利用 Tailwind CSS 配合 React 實現兼具童趣、迷因風格且高互動性的 RWD 網頁。
>
> **Task:** 請為一款名為「義次元腦洞大開」的國小校園大地遊戲撰寫 UI 組件與主要頁面結構。
> 
> **Design References & Styles:**
> 1. 請讀取並解析我提供的圖片 `ui 參考.png`，嚴格遵守圖中的色彩計畫配置（包含主色調、背景點綴、按鈕陰影風格等），建立一套 Tailwind Theme 設定。
> 2. 介面需具有「立體、圓角、童趣」的特質，按鈕需要有明顯的點擊回饋 (Active/Hover effects)，類似於現代手機遊戲的 UI 質感。
> 3. 字體預設採用 `JasonHandwriting2-Regular`（清松手寫體），請在 CSS 中處理字體的加載。
>
> **Core Components Required:**
> 1. **BottomNavigationBar**: 底部需有包含「背包狀態」與「相機掃描」的立體浮動選單，掃描按鈕應位於中央且放大突出。
> 2. **Dialog & Audio Panel**: 顯示角色對話的氣泡框，並內建簡潔的「播放音軌」按鈕與文字打字機特效 (Typewriter effect)。打字機特效需配合音檔長度稍微控制。
> 3. **ItemCard**: 一個能展示食材與道具的卡片組件，具有微小的懸浮動畫 (`animate-bounce` 或自定義的上浮特效)。
> 4. **Task Interactive UI**: 
>    - 實作一個支援「拖曳排序 (Drag & Drop)」或「數字輸入鍵盤」的通用容器，讓不同的關卡組件可以掛載於其中。
>
> **Constraints:**
> - 使用 React 函數式組件 (Functional Components) 與 hooks。
> - 所有樣式必須使用 Tailwind CSS classes 完成，不要撰寫額外的 CSS 檔（字體 @font-face 引入除外）。
> - 確保在手機螢幕 (max-width: 768px) 下排版不會擁擠，按鈕大小至少要有 48px 以符合觸控熱區標準。
>
> 請先給出整體架構的 `Tailwind Config` 設定與最核心的 `MainDashboard` (主畫面背包) 及 `LevelTaskContainer` (關卡挑戰容器) 的程式碼實作。
